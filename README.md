## Features

*   **File Browser:** A tree-based file browser to navigate your projects.
*   **Code Editor:** A powerful code editor with syntax highlighting for many languages, powered by Ace Editor.
*   **Multiple Workspaces:** Create and manage multiple workspaces to organize your projects.
*   **User Management:** A simple user management system with admin roles and permissions.
*   **API:** A basic API to interact with the IDE programmatically.
*   **Installer:** A simple web-based installer to get you up and running quickly.

## Requirements

*   A web server (e.g., Apache, Nginx)
*   PHP 7.4 or higher
*   **Optional (to unlock full AI Agent capabilities):**
    *   **Node.js**: Allows the AI to run build tools, start dev/preview servers, and run JS-based scripts (like Puppeteer).
    *   **Docker**: Allows the AI to execute safe testing and run containerized services without affecting your host system.
    *   **ImageMagick**: Allows the AI to perform image optimizations, compression, and visual assets processing.

## Installation

1.  **Download:** Download the latest release or clone the repository to your web server.
2.  **Permissions:** Make sure the web server has write permissions to the `data` directory.
3.  **Run Installer:** Open your web browser and navigate to `http://your-domain.com/install.php`.
4.  **Follow Instructions:** The installer will guide you through the process of creating a `config.php` file, setting up an admin user, and creating your first workspace.

    *   **Root Path:** The absolute path to the root of the IDE application on your server. The installer will try to pre-fill this for you.
    *   **Data Path:** The absolute path to the `data` directory. This is where all your configuration, users, and workspaces will be stored.
    *   **Application URL:** The URL of your IDE application.
    *   **Admin User:** Create your admin user account.
    *   **First Workspace:** Create your first workspace. You can specify a name and a path for it.

5.  **Done:** Once the installation is complete, you will be redirected to the IDE. You can now log in with your admin credentials.

## Configuration

The main configuration file for the IDE is `config.php`. This file is generated by the installer, but you can also create it manually.

Here are the available options:

*   `ROOT_PATH`: The absolute path to the root of the IDE application.
*   `DATA_PATH`: The absolute path to the `data` directory.
*   `APP_URL`: The full URL of your IDE application.
*   `DEBUG`: Set to `true` to enable error reporting for development, or `false` for production.

## Usage

### Logging In

Navigate to the `login.php` page to log in. Use the credentials you created during installation.

### Workspaces

Workspaces are used to organize your projects. You can create multiple workspaces and switch between them. Each workspace has a name and a path to a directory on your server.

### File Browser

The file browser on the left side of the screen allows you to navigate the files and directories in your current workspace. You can create, rename, and delete files and directories.

### Code Editor

The code editor is powered by Ace Editor and supports syntax highlighting for a wide variety of languages. You can customize the editor's theme and settings from the editor's settings menu.

## Permissions

The IDE has a simple permission system based on the `data/users/users.php` file.

Each user has a `permissions` array. The `*` permission gives the user full access to all features, including admin features.

Example of a user with admin permissions:
```php
return [
  'admin' => 
  [
    'password' => '$2y$12$NHC/hM.c5LY0dpou/6SXYO3vv9PLgQwZNcvIVAAYPPw0hSA3A/JqK',
    'permissions' => 
     [
      0 => '*',
     ],
  ]
];
```

## API

The IDE provides a basic API through the `api.php` file. You can use the API to perform actions like listing files, reading file content, and saving files.

To use the API, you need to send a POST request to `api.php` with the following parameters:

*   `action`: The action you want to perform (e.g., `get_files`, `get_content`, `save_content`).
*   `...other_params`: Other parameters depending on the action.

You will need to be authenticated to use the API.
